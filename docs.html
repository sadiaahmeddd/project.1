Pokédex API Documentation

Main Client | Endpoint Tester

Base URL

http://localhost:3000/api

All endpoints return JSON and support GET and HEAD. POST requests accept either JSON or x-www-form-urlencoded bodies.

GET /api/pokemon

Returns a list of Pokémon. Optional query parameters:

  • name – partial match (case-insensitive)
  • type – exact match (e.g. Fire)
  • limit – limit number of results

Example:

GET /api/pokemon?type=Fire&limit=3
→ { "count": 3, "data": [ ... ] }

POST /api/pokemon

Adds a new Pokémon. Requires "name" and "type".

Example body:

{
  "name": "Testmon",
  "type": ["Normal"]
}

Response (201): The created Pokémon object.

GET /api/pokemon/:id

Returns a single Pokémon by ID.

Example:

GET /api/pokemon/25
→ { "id": 25, "name": "Pikachu", "type": ["Electric"] }

POST /api/pokemon/:id

Edits a Pokémon’s details. Partial update of fields such as name, height, weight, or weaknesses.

Example body:

{ "height": "0.50 m" }

Response: 204 No Content on success.

GET /api/types

Returns a list of all Pokémon types.

Example:

GET /api/types
→ { "data": ["Fire","Water","Grass", ...] }

GET /api/weaknesses

Returns a list of all Pokémon weaknesses.

Example:

GET /api/weaknesses
→ { "data": ["Rock","Ice","Fire", ...] }

GET /api/random

Returns one random Pokémon.

Example:

GET /api/random
→ { "id": 4, "name": "Charmander", "type": ["Fire"] }

POST /api/delete/:id

Deletes a Pokémon from memory (soft delete). No response body.

Example:

POST /api/delete/25
→ 204 No Content

POST /api/save

Saves the current dataset to data/pokedex.runtime.json.

Example:

POST /api/save
→ { "saved": true, "path": "data/pokedex.runtime.json" }
